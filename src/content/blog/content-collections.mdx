Astro の [Content Collections](https://docs.astro.build/ja/guides/content-collections/) を使うと、Markdown / MDX ファイルに型が付いて非常に快適です。

## どういう仕組み？

`src/content.config.ts` に Zod でスキーマを定義すると、フロントマターに型推論が効くようになります。

```typescript
// src/content.config.ts
import { defineCollection, z } from "astro:content";

const blog = defineCollection({
  type: "content",
  schema: z.object({
    title: z.string(),
    description: z.string(),
    pubDate: z.coerce.date(),
    categories: z.array(z.string()).default([]),
  }),
});

export const collections = { blog };
```

## 使い方

ページ側では `getCollection` でコンテンツを取得し、`getStaticPaths` で動的ルートを生成します。

```typescript
import { getCollection } from "astro:content";

const posts = await getCollection("blog");
```

型安全なので、フロントマターのフィールド名を間違えるとビルド時にエラーが出ます。地味にありがたい。

## MDX の便利さ

MDX にしておくと Astro コンポーネントをそのまま埋め込めます。
将来的に独自コンポーネントを使いたくなったときのための布石です。
